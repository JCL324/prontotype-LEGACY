Pages and Layouts

Adding a file with a .html extension to the /site/pages directory create a page in your prototype. You can obviously organise these into folders as you would a normal site, and Prontotype will make sure that that are routed to using the pages directory as the 'web root'.

For instance, a file at /site/pages/about/company.html will be accessible via the URL http://example.com/about/company

You can also define layouts in the /site/layouts directory that your pages can inhertit from. These layouts define blocks that are replaced by content in your page files. See the documentation for more information on how this works.

Ordering pages

To keep your files ordered nicely in your directories (and to control order that they are output from things like the navigation macros), you can also prefix the filename with a number followed by a dash, underscore or full stop (period). This ordering information will not show up in the URL.

So using the above example again, but including some ordering information, a file at /site/pages/about/1-company.html will still be accessible at http://example.com/about/company and will be the first file in the 'about' section.

Page IDs

Lastly, you can give pages unique IDs, and use these to build links to pages so that if you move the pages around (which happens a lot when prototyping!) you won't have to update all your links.

You can give a page and ID by adding an alphanumeric string (underscores and dashes are ok in there too) surrounded by square brackets into the filename. You can also follow this by a dash, underscore or full stop (period) to help seperate it from the rest of the filename, of you wish.

The ID string, like the ordering prefix, will not show up in the URL. So a page at /site/pages/about/[c123]-company.html will have a page ID of c123 and will be shown at the URL http://example.com/about/company

Once your page has an ID, you can always print it's current URL into your template by using:

{% raw %}{{ pages.get_by_id('PAGE-ID').url }}{% endraw %}

Where PAGE-ID is the ID of the page you are interested in.

Bringing it all togther

You can of course mix ordering with pages ids - so combining the examples above we would have a page that sits at /site/pages/about/1-[c123]-company.html which is found at the URL http://example.com/about/company, is the first page in the 'about' section and has a page ID of c123
