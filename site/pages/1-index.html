Getting Started with Prontotype

Prontotype is set up and ready for you to get prototyping!

We've included a few example pages and files to help you understand a little bit more about how Prontotype works. You can go ahead and delete or replace them as soon as you are ready.

Example files you'll want to remove:

  • /actions/example.php
  • /data/example.yml
  • /public/example.less
  • /site/layouts/example-base.html
  • /pages/1-index.html
  • /pages/2-pages-and-layouts.html
  • /pages/3-macros-examples.html
  • /pages/4-using-data.html
  • /pages/5-404.html

There is also some example data in /data/users.yml and /data/notes.yml which you can clear out, but it's best to leave the files themselves there as they are used internally by the system.

The Prontotype toolbar

For the example site the Prontotype toolbar has been enabled - you should see it down in the bottom right had side of your browser window.

Try toggling the notes view or logging in as one of the users (defined in the /data/users.yml file) to get a feel for how some of the more dynamic aspects of Prontotype can work.

Learn More…

The pages in this example site don't by any means show you everything that you can do with Prontotype. You should check out the full documentation to learn more about working with user logins, available template tags, LessCSS support and more.
